## 总结文档

​		经过7天工作日的学习和实践，基本完成了entrytask要求的功能，也满足了基本的性能需求，在安全性上欠缺考虑。在项目实现的过程中，熟悉了go的基本语法，对go的web项目开发有了一定的认识。

- 明白了如何用go的标准库去进行web server 的开发，学习到了go web server是如何调用静态资源，如何与静态页面进行交互。

- 明白了如何在go tcp client 和 go tcp server之间的进行交互，学习到了go socket通信、rpc机制、tcp链接池的实现。

- 学习到了go如何与redis进行交互，go如何与mysql进行交互，2种链接池的运用。	

  

​        不仅学习到了项目开发的方法，也学习到了项目开发的思路和对性能的探索，前期项目功能点实现时并不能满足性能的需求，在导师的帮助下，查阅相关资料学习到了运用链接池的好处。在实现功能逻辑细节的重构后加上链接池的使用，性能已经满足了项目的要求。

​		通过本次项目的实践，不仅在技术上学习到了很多新的知识，在项目开发的思维上也有了新的认识，项目的开发不仅仅在于功能点的实现，更多的是在于代码的结构和规范、性能的探索，更要有丰富的文档进行理论支撑，希望在以后的工作中不能仅满足于学生时对功能实现的追求，更注意项目的个方面的质量。
